# CPU Cooler LCD Setup

Displays CPU/GPU temps and usage on the digital screen of a
**Thermalright Assassin X 120R Digital** cooler.

## What this is

Based on the open-source project
[MathieuxHugo/digital_thermal_right_lcd](https://github.com/MathieuxHugo/digital_thermal_right_lcd)
(GPL-3.0), copied into this repo (`src/`, `conf/`, `config.json`, etc.).
It talks to the cooler's display over USB HID.

- Device: `0416:8001` (Winbond Electronics Corp. HID Transfer)
- Layout config: `conf/tr_assassin_x_120r.json` ("TR Assassin X 120R")
- Active config: `conf/config.json`

## How it's run

A systemd **user** service runs the controller continuously:

- Unit file: `~/.config/systemd/user/digital-lcd-controller.service`
- Runs: `.venv/bin/python -u src/controller.py conf`
- Enabled + started, auto-restarts on failure
- Starts on graphical login (not before login / headless boot, unless
  `loginctl enable-linger paul` is run)

Useful commands:

```bash
systemctl --user status digital-lcd-controller    # check status
systemctl --user restart digital-lcd-controller   # restart after config edit
journalctl --user -u digital-lcd-controller -f    # view logs
```

## System changes made

- Created `/etc/udev/rules.d/99-thermalright-lcd.rules` (sudo) so the
  device (`idVendor=0416`, `idProduct=8001`) is readable/writable by
  normal users (`MODE="0666"`) on both the `usb` and `hidraw` subsystems.
- Reloaded udev rules with `udevadm control --reload-rules` and
  `udevadm trigger`.

## Python environment

- Virtualenv at `.venv/` (Python 3.12), created with
  `--system-site-packages` so it can use the system `gi`/PyGObject and
  `tkinter` bindings (these aren't easily pip-installable - they need
  apt packages `python3-tk`, `python3-gi`, `python3-gi-cairo`,
  `gir1.2-gtk-3.0`, `gir1.2-ayatanaappindicator3-0.1`, all already
  present on this system).
- Dependencies in `requirements.txt`: `hid`, `numpy`, `psutil`,
  `pyamdgpuinfo`, `pynvml`
  (Windows-only deps `WinTmp`, `WMI`, and `pyinstaller` were dropped from
  the original requirements list since this runs on Linux)
- Also pip-installed (not in requirements.txt): `pystray`, `pillow`
  (used by the tray icon, see below)

## Configuration

Edit `conf/config.json` (the running service picks up changes
automatically each loop, no restart needed... but `restart` is safe if
something looks stuck):

- `display_mode`: `"alternate_metrics"` (current) cycles through
  `cpu_temp` -> `cpu_usage` -> `gpu_temp` -> `gpu_usage`.
  Set to `"cpu_temp"` to show CPU temperature only, or
  `"alternate_temps"` to cycle only `cpu_temp` <-> `gpu_temp` (added to
  `conf/tr_assassin_x_120r.json`'s `display_modes`, also selectable from
  the GUI's "Choose display mode" dropdown).
- `cpu_temperature_unit` / `gpu_temperature_unit`: `"celsius"` or
  `"fahrenheit"`.
- `metrics.colors`, `cycle_duration`, `update_interval`: display
  colors/timing.

## Hardware detected

- GPU: NVIDIA GeForce RTX 5070 (temps/usage read via `nvidia-smi`/pynvml)
- CPU temp: read via `psutil` (lm-sensors)

## GUI for adjusting settings

`src/led_display_ui.py` is a tkinter GUI (already part of the upstream
project) for changing layout, display mode, colors, temp units, and
thresholds. It writes directly to `conf/config.json`, which the running
service picks up automatically (no restart needed).

- Requires the `python3-tk` system package (installed via apt - not
  pip-installable).
- Launch via the desktop entry "CPU Cooler LCD Config" (added to
  `~/.local/share/applications/cpu-cooler-lcd-config.desktop`), or run:

```bash
.venv/bin/python src/led_display_ui.py conf/config.json
```

### Display power toggle

A "Display Power" button (top of the GUI) writes `"enabled": true/false`
to `conf/config.json`. When `enabled` is `false`, the controller (still
running as a service) sends an all-LEDs-off frame each cycle instead of
metrics, so the screen goes blank without stopping the service. Toggling
it back on resumes normal display immediately.

### Restart Tray Icon / Unload Everything buttons

Two buttons next to "Reset default config":

- **Restart Tray Icon** - kills any running `tray_icon.py` process,
  starts the `digital-lcd-controller` systemd service (in case it was
  stopped), and relaunches the tray icon. Use this after the tray icon
  has been quit, or to recover after "Unload Everything".
- **Unload Everything** - kills the tray icon process and stops the
  `digital-lcd-controller` systemd service, fully shutting down the LCD
  display and tray icon (the config GUI itself stays open).

### Window icon

A thermometer icon (`assets/icon.png`, generated by
`assets/generate_icon.py`) is used as the GUI window/taskbar icon and in
both desktop entries (`cpu-cooler-lcd-config.desktop` and
`cpu-cooler-tray.desktop`).

## Tray icon

`src/tray_icon.py` adds an icon to the system tray (next to the clock,
Cinnamon panel) showing the live CPU temperature. Built with `pystray`
using the AyatanaAppIndicator backend (`gi`/GTK).

- Click the icon for a menu:
  - **Display On** (checkbox) - same as the GUI's power toggle, writes
    `"enabled"` to `conf/config.json`.
  - **Open Settings** - launches `led_display_ui.py`.
  - **Quit Tray Icon** - closes just the tray icon (the LCD controller
    service keeps running). To bring it back, use the "Restart Tray
    Icon" button in the Settings GUI.
- Autostarts at login via
  `~/.config/autostart/cpu-cooler-tray.desktop`.
- Run manually with:

```bash
.venv/bin/python src/tray_icon.py
```

### Known issue fixed

The GUI's "Reset default config" button originally reset
`layout_mode` to `"Pearless Assasin 120"` (a 124-LED layout), which
doesn't match this 31-LED Assassin X 120R and produced garbled output
on the cooler's screen. Fixed by changing the `default_config` in
`src/config.py` to use `layout_mode: "TR Assassin X 120R"` and
`display_mode: "alternate_metrics"`. If you ever click "Reset default
config" and the screen looks wrong again, edit `conf/config.json`'s
`layout_mode` back to `"TR Assassin X 120R"` and run
`systemctl --user restart digital-lcd-controller`.

## Building the .deb package

The `debian/` directory contains a native Debian package (`cpu-cooler-lcd`)
that installs the controller, settings GUI, and tray icon as system
commands, with all Python dependencies pulled from apt (no venv needed).

Build tools (one-time):

```bash
sudo apt install debhelper dh-python
```

Build:

```bash
dpkg-buildpackage -us -uc -b
```

This produces `../cpu-cooler-lcd_1.0-1_all.deb`.

## Installing the .deb

```bash
sudo apt install ./cpu-cooler-lcd_1.0-1_all.deb
```

This installs:

- `/usr/bin/cpu-cooler-lcd-controller`, `-config`, `-tray` - the three
  commands (controller daemon, settings GUI, tray icon)
- `/usr/lib/systemd/user/cpu-cooler-lcd.service` - the controller service
- `/usr/share/applications/cpu-cooler-lcd-config.desktop` and
  `/etc/xdg/autostart/cpu-cooler-lcd-tray.desktop` - launcher/autostart
  entries
- `/lib/udev/rules.d/60-cpu-cooler-lcd.rules` - device permissions for
  `0416:8001`
- A user config seeded at `~/.config/cpu-cooler-lcd/config.json` on first
  run (from `src/config.py`'s `default_config`)

After installing, replug the cooler's USB cable (or run
`sudo udevadm control --reload-rules && sudo udevadm trigger`) so the new
udev rule takes effect, then enable the service:

```bash
systemctl --user enable --now cpu-cooler-lcd
```

The settings GUI and tray icon can be launched from the application menu,
or directly with `cpu-cooler-lcd-config` / `cpu-cooler-lcd-tray`.

Note: this machine's existing manual venv-based setup
(`~/.config/systemd/user/digital-lcd-controller.service`, `.venv`, etc.,
described above) continues to run independently of the `.deb`; the two
aren't installed at the same time on this machine.
